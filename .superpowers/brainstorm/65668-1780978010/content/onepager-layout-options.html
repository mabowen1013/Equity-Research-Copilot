One-Pager 页面布局方向

目标：输入 ticker 后快速生成可引用的一页式研究报告，同时保留数据准备状态和 evidence trail。

A. Report-first
Ticker input | Generate | Status
Executive Summary
Financial Snapshot
Anomaly Highlights
MD&A / Risk / 8-K Highlights
Bull / Bear Thesis
Citations + Warnings

A. Report-first 一页阅读流

最像真正 One-Pager：报告内容是主角，状态和证据折叠在下方。推荐作为 v1。

B. Research workbench

更适合调试和展示 agent 步骤，但视觉上不像“一页报告”，v1 会显得偏工程工具。

C. Tabbed Report
Input | Report | Evidence | Diagnostics
Active Tab: Report
Financials
Qualitative
8-K Events
Thesis + Citations

C. Tabs with report preview

平衡展示和调试，但“one-page”感觉会被 tabs 打散，适合 v2 扩展。

我的建议

选 A：报告优先，一屏内展示生成入口、核心摘要、财务异常、定性洞察和多空逻辑；filing universe、job 状态、trace 和 citations 做成可展开区域。
